# クリーチャービジュアル v2 設計（Issue #6）

日付: 2026-07-07
ステータス: ユーザー承認済み（モックアップで全5種族の見た目を確認済み）
モックアップ: https://claude.ai/code/artifact/18f98a97-ae12-4e4f-a073-79ee01506aeb
（プレビューHTML実体: 同ディレクトリの `2026-07-07-creature-v2-mockup.html`。
実装時はこのモックアップ内の v2 生成コード（slimeV2 / beastV2 / bipedV2 /
eyeTentacleV2 / orbV2 と共通ヘルパー）を TypeScript に移植する）

## 背景と課題

Phase 2 の手続き型クリーチャーは「基本図形（丸・楕円・矩形）の寄せ集め」で、
以下のフィードバックがあった:

- かわいらしさ（愛嬌）がない
- スライムがスライムに見えない
- けものがチープ
- 同じ種族だとほぼ同じに見える（複雑性の不足）
- 単純図形ベースがチープさの根本原因

## 方針（承認済み）

**パーツライブラリ合成方式＋一筆シルエット**。
人間が関わるのは開発時のパーツ（形状テンプレート）設計のみで、
実行時の生成は従来どおり漢字DNAから完全に決定論的・機械的に行う。

### 全種族共通の文法

1. **一筆シルエット**: 体の輪郭は1本の閉じたベジェパス。図形の寄せ集めにしない
2. **つややかな目**: 白目＋大きな瞳＋ハイライト2つ（かわいらしさの核）。
   にこにこ時はほっぺの赤み。怒り時は内側が下がる太眉
3. **3層陰影**: おなかの明るみ（belly）／下部の暗いバンド（dark, clipPath使用）／頭・体のツヤ（白, 低不透明度）
4. **縁取り**: 主要形状に暗色アウトライン（stroke-width 2〜2.6, stroke-linejoin round）
5. **接地/浮遊の影**: 地上種は接地影、浮遊種は小さく薄い影を下方に離して描く
6. **ポーズ差**: 向きの左右反転・傾き・スタンス等を DNA から決定

### カラーパレット（hue は DNA 由来）

```
main:    hsl(hue, 68%, 55%)   体色
belly:   hsl(hue, 60%, 76%)   おなか・明部
dark:    hsl(hue, 60%, 36%)   ツノ・翼・暗部
outline: hsl(hue, 55%, 15%)   縁取り・顔パーツ
accent:  hsl((hue+180)%360, 75%, 60%)  きらめき等
ほっぺ:  hsl(350, 85%, 72%) opacity 0.5（固定）
```

## 種族別パーツと DNA マッピング

パーツ選択はすべて `(整数化したDNA値の和) % バリアント数` 形式で決定論的に行う。
連続値（サイズ・傾き・視線など）は DNA 値をそのまま線形変換する。

### 種族1: スライム（しずく型・底面フラット）

| 要素 | バリアント/制御 | DNA |
|---|---|---|
| 体型 | クラシック / とろけ（左右に水たまり＋伝う雫）/ とげ背 | hRatio 3分割 |
| つぶれ具合 | 横潰れ⇄縦伸び 3段階（W/H を逆相関で変形） | strokeCount % 3 |
| 先端 | 高さ・傾き（カールは💩に見えるため禁止） | curvature |
| 目 | にこにこ（＋ほっぺ）/ 怒り眉 | symmetry > 0.55 |
| 口 | にゃんこ口 / にやり牙 / ぽかん | (strokeCount + round(hRatio*10)) % 3 |

底面は全バリアント共通でフラット（角丸で接地）。波々の底は使わない。

### 種族3: けもの（大頭身の四足獣）

| 要素 | バリアント | DNA |
|---|---|---|
| 耳/ツノ | とがり耳 / 曲がりヅノ / まる耳 | (sc + round(sym*10)) % 3 |
| たてがみ | 有無（頭の後ろに三角形ファン） | curvature > 0.5 |
| 尻尾 | ふさ尻尾 / とげ尻尾 | symmetry > 0.5 |
| スタンス | 直立 / 座り / 駆け出し | (sc + round(cur*10) + round(sym*10)) % 3 |
| 向き | 左右反転 | (sc + round(hR*10) + round(hue/30)) % 2 |
| 首の傾き | ±7° | symmetry |

- 座り: 胴体を rotate(-25°) でお尻を接地、後脚は畳んだもも楕円＋前向き足先、前脚のみ直立、尻尾も接地位置へ移動
- 駆け出し: 足先を前後に ±14px（手前側）開く
- 遠い側の脚は dark 塗りで奥行き表現

### 種族0: にそくほこう（大頭身のこびと/ゴブリン）

| 要素 | バリアント | DNA |
|---|---|---|
| 体型 | 洋ナシ型の幅（連続変化） | hRatio |
| ツノ/毛 | 2本ヅノ / 1本ヅノ / とんがり毛3本 | (sc + round(sym*10)) % 3 |
| ポーズ（腕） | 両手下ろし / 片手上げ / ファイティング | (sc + round(cur*10) + round(sym*10)) % 3 |
| 口 | 下牙にやり / にこ / ぽかん | (sc + round(hR*10)) % 3 |
| 目 | にこにこ（＋ほっぺ）/ 怒り眉 | symmetry > 0.55 |
| 向き | 左右反転 | (sc + round(hR*10) + round(hue/30)) % 2 |

とがり耳は全個体共通。腕は「アウトライン色の太ストローク＋本体色ストローク」の
二重描きで縁取り表現、手先はこぶし円。

### 種族2: しょくしゅがん（浮遊する一つ目）

| 要素 | バリアント/制御 | DNA |
|---|---|---|
| 瞳 | まる / 縦スリット | symmetry > 0.55 |
| まぶた | ぱっちり / 怒りまぶた / にっこり半目 | (sc + round(hR*10)) % 3 |
| まつ毛 | 有無（上側3本） | symmetry > 0.6 |
| 触手 | 4〜7本。先細りの塗りパス、先端が交互にカール | sc % 4, curvature |
| 視線 | 瞳の向き（x: hRatio, y: curvature） | 連続 |
| 浮遊高さ | 中心 y ±2.5px | curvature |

### 種族4: ふゆうたい

| 要素 | バリアント | DNA |
|---|---|---|
| 体 | まんまる玉 / おばけ（ドーム＋スカート）/ ほのお（しずく＋内炎） | (sc + round(sym*10)) % 3 |
| 翼 | なし / コウモリ翼 / ちび羽 | (round(hR*10) + round(cur*10)) % 3 |
| 口 | にやり牙 / にこ / ぽかん | (sc + round(hR*10)) % 3 |
| 傾き | ±6° | symmetry |
| 向き | 左右反転 | (sc + round(hR*10) + round(hue/30)) % 2 |

オーラ（破線円2重）ときらめき（accent 色の菱形3つ）は現行から継承。

## アーキテクチャ

### ファイル構成

`src/logic/creatureGenerator.ts`（現在307行）は5種族で肥大化するため分割する:

```
src/logic/creatures/
  palette.ts        colors2(hue) ほか色定義
  parts.ts          共通パーツ（glossyEye, browPair, grinMouth, shadow, 一筆パスヘルパー）
  slime.ts          generateSlime(dna)
  beast.ts          generateBeast(dna)
  biped.ts          generateBiped(dna)
  eyeTentacle.ts    generateEyeTentacle(dna)
  orb.ts            generateOrb(dna)
  index.ts          selectSpecies / generateCreatureName / generateCreature（公開API）
src/logic/creatureGenerator.ts   → creatures/index.ts の再エクスポート（既存 import を壊さない）
```

### 公開 API・互換性（変更しない）

- `generateCreature(dna, word): CreatureSpec`、`selectSpecies`、`generateCreatureName` のシグネチャ維持
- 決定性維持: 同じ DNA・単語 → 同じ SVG（乱数・Date 等は使わない）
- SVG は `width="120" height="120" viewBox="0 0 120 120"` を維持（EnemyDisplay 側は変更不要）
- 種族の割り当てロジック（selectSpecies）・名前は変更しない
- 既存テスト `creatureGenerator.test.ts` はそのまま通ること

### 注意点

- **clipPath の id 衝突**: 陰影に clipPath を使うため、モジュールレベルのカウンターで
  一意な id を発行する（同一 DOM に複数クリーチャーが存在しても衝突しない）。
  ただし id が変わっても SVG の見た目は同一なので決定性テストは
  「id を正規化して比較」または「同一入力で id カウンターも同じ進み方をする」設計にする
  （推奨: 生成1回につき id を `c${species}-${内部連番}` とし、決定性テストは id 部分を除いて比較）
- 描画定数（座標・サイズ）は各種族ファイル先頭の定数オブジェクトにまとめる（CLAUDE.md 準拠）
- パーツ選択式はモックアップ準拠。実装時に変更しない（見た目の承認が無効になるため）
